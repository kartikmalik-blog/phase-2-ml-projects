CELL 3 — FEATURE ENGINEERING (THE PHASE 2 POWER!)
================================================================

WHY FEATURE ENGINEERING HERE:
    Raw brightness values (u, g, r, i, z) are useful
    BUT colour DIFFERENCES between bands are MORE useful!
    Astronomers have known this for decades!
    We encode this domain knowledge into features!

ENGINEERED FEATURES EXPLAINED:

u_g = df['u'] - df['g']  (UV minus Green)
    → Colour index between ultraviolet and green bands
    → Different objects have different UV vs green profiles
    → Stars: blue-white → high u_g
    → Galaxies: mixed → moderate u_g
    → QSOs: extreme energy → very distinct u_g

g_r = df['g'] - df['r']  (Green minus Red)
    → Colour index between green and red bands
    → Object temperature affects this ratio
    → Hot objects (stars) → bluer → different g_r than cool galaxies

r_i = df['r'] - df['i']  (Red minus Near-IR)
    → Near-infrared colour index
    → Very useful for separating cool objects

i_z = df['i'] - df['z']  (Near-IR minus IR)
    → Infrared colour index
    → Captures dust and gas emission differences

u_r = df['u'] - df['r']  (UV to Red span)
    → Wide colour baseline
    → Captures overall spectral slope

g_z = df['g'] - df['z']  (Green to IR span)
    → Widest colour baseline
    → Most powerful colour separator!
    → Became #6 most important feature!

log_redshift = np.log1p(np.abs(df['redshift']))
    → Log transform of redshift
    → Redshift very skewed → log makes it more normal
    → QSOs have redshift 1.7 vs Stars ~0
    → Became #2 most important feature!

redshift_sq = df['redshift'] ** 2
    → Squared redshift
    → Captures non-linear redshift patterns
    → Became #3 most important feature!

high_redshift = (df['redshift'] > 0.5).astype(int)
    → Binary flag: is redshift > 0.5?
    → QSOs almost always > 0.5
    → Stars almost always < 0.01
    → BECAME #1 MOST IMPORTANT FEATURE (32.41%!)

sky_area = df['alpha'] * df['delta']
    → Interaction of sky position coordinates
    → Captures location-dependent patterns

total_brightness = (u+g+r+i+z) / 5
    → Average brightness across all bands
    → Stars tend to be brighter than distant galaxies

brightness_range = max(u,g,r,i,z) - min(u,g,r,i,z)
    → Spread of brightness across bands
    → Objects with unusual spectra = large range

CELL 4 — BUILD & COMPARE ALL MODELS
================================================================

THE MULTICLASS PROBLEM:
    3 classes → multiclass classification!
    AUC uses: roc_auc_score(y_test, probs,
                multi_class='ovr', average='macro')
    'ovr' = One vs Rest (each class vs all others)
    'macro' = average AUC across all 3 classes equally

5 MODELS COMPARED:
    Logistic Regression → 95.44% accuracy, AUC=0.986
    SVM (RBF)           → 96.69% accuracy, AUC=0.991
    Neural Network      → 97.30% accuracy, AUC=0.994
    Random Forest       → 98.08% accuracy, AUC=0.996
    XGBoost             → 97.88% accuracy, AUC=0.9964 🏆

WHY XGBOOST WINS:
    Highest AUC (0.9964)!
    Best generalisation (smallest gap)
    Built-in regularisation prevents overfit
    Fast on 100k samples

ASTRONOMICAL MEANING:
    Redshift dominates because:
    Stars  → redshift ≈ 0 (in our galaxy, nearby)
    Galaxies → redshift ≈ 0.4 (moderate distance)
    QSOs  → redshift ≈ 1.7 (extremely far away!)

    Our high_redshift flag perfectly captures this!
    QSO if redshift > 0.5 → 32% of all predictive power!
